# 🏨 Gapo Hotel Online Booking System - System Analysis & Design

This repository contains the comprehensive **System Analysis and Design** documentation for the Gapo Hotel Online Booking System. 

Although this repository does not contain source code, it demonstrates a deep understanding of the **Software Development Life Cycle (SDLC)**, specifically focusing on the Planning, Analysis, and Design phases.

## 📌 Project Overview
The objective of this project was to design an end-to-end online booking and payment system for Gapo Hotel, aiming to solve the inefficiencies of manual walk-in bookings and lack of online payment tracking.

## 🛠️ Key Skills & Concepts Demonstrated
This documentation showcases my ability to design scalable software architecture and communicate system logic using industry-standard modeling tools.

### 1. Unified Modeling Language (UML) Diagrams
* **Use Case Diagrams & Descriptions:** Defining system actors (Admin, Manager, Customer, Housekeeper) and their interactions with the system.
* **Sequence & Activity Diagrams:** Mapping out the logical flow for core features like User Registration, Room Booking, and Payment Processing.
* *Class Diagram:** Designing the object-oriented structure, including relationships between `Booking`, `Room`, `Customer`, and `Payment` classes.
* **State Diagram:** Illustrating the lifecycle of a booking process from 'In Process' to 'Reserved'[cite: 880].
* **Deployment Diagram:** Outlining the hardware and software architecture mapping (Client Workstation, Web Server, Database Server).

### 2. Database & System Architecture
* *Database Logic Subsystem:** Designing primary and backup database architectures for data redundancy.
* **Persistent Data Design:** Detailed database schema design including Primary Keys, Foreign Keys, and Data Types for all system entities.
* **Presentation Logic:** Mapping the navigation and relationship between different user interface forms.

### 3. Security & Control Design
* Designed system controls including **Input Integrity Controls** (Data Validation) and **Database Integrity Controls** (Access Control, Data Encryption) to ensure system security and data accuracy.
