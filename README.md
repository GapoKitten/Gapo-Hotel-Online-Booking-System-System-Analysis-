# 🏨 Gapo Hotel Online Booking System - System Analysis & Design

[cite_start]This repository contains the comprehensive **System Analysis and Design** documentation for the Gapo Hotel Online Booking System[cite: 186]. 

[cite_start]Although this repository does not contain source code, it demonstrates a deep understanding of the **Software Development Life Cycle (SDLC)**, specifically focusing on the Planning, Analysis, and Design phases[cite: 306, 308].

## 📌 Project Overview
[cite_start]The objective of this project was to design an end-to-end online booking and payment system for Gapo Hotel, aiming to solve the inefficiencies of manual walk-in bookings and lack of online payment tracking[cite: 209].

## 🛠️ Key Skills & Concepts Demonstrated
This documentation showcases my ability to design scalable software architecture and communicate system logic using industry-standard modeling tools.

### 1. Unified Modeling Language (UML) Diagrams
* [cite_start]**Use Case Diagrams & Descriptions:** Defining system actors (Admin, Manager, Customer, Housekeeper) and their interactions with the system[cite: 412, 544].
* [cite_start]**Sequence & Activity Diagrams:** Mapping out the logical flow for core features like User Registration, Room Booking, and Payment Processing[cite: 624, 721].
* [cite_start]**Class Diagram:** Designing the object-oriented structure, including relationships between `Booking`, `Room`, `Customer`, and `Payment` classes[cite: 924].
* [cite_start]**State Diagram:** Illustrating the lifecycle of a booking process from 'In Process' to 'Reserved'[cite: 880].
* [cite_start]**Deployment Diagram:** Outlining the hardware and software architecture mapping (Client Workstation, Web Server, Database Server)[cite: 1131].

### 2. Database & System Architecture
* [cite_start]**Database Logic Subsystem:** Designing primary and backup database architectures for data redundancy[cite: 1120].
* [cite_start]**Persistent Data Design:** Detailed database schema design including Primary Keys, Foreign Keys, and Data Types for all system entities[cite: 1148].
* [cite_start]**Presentation Logic:** Mapping the navigation and relationship between different user interface forms[cite: 1037].

### 3. Security & Control Design
* [cite_start]Designed system controls including **Input Integrity Controls** (Data Validation) and **Database Integrity Controls** (Access Control, Data Encryption) to ensure system security and data accuracy[cite: 1243, 1251, 1256].
